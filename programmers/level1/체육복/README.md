**어려웠던 점**
- 그리디를 사용할 때 **정렬된 상태**의 리스트여야 원하는 대로 놓치는 부분 없이 최적의 방법을 찾을 수 있다는 점을 놓쳐 조금 헤멨다.
lost와 reserve를 정렬된 상태가 아닌 채로 앞/뒤 사람에게 빌려주는 방식으로 진행하면 이미 뒷 번호가 앞의 사람한테 빌려줬을 경우 reserve에 있는 또다른 앞의 사람이 그 누구에게도 빌려줄 수 없는 상태가 되게 된다.
